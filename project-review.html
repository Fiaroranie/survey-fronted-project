Survey Frontend 项目结构与关键代码说明

这份 HTML 文档用于快速理解当前问卷项目：前端 Vue 表单如何组织，数据结构在哪里定义，提交接口如何调用，后端如何写入 PostgreSQL。

项目：survey-frontend 前端：Vue 3 + Vite 后端：Express + PostgreSQL 入口：src/App.vue

1. 项目总览

前端职责

展示问卷、维护表单状态、做基础校验、调用提交接口。

后端职责

接收问卷 JSON，在事务中写入多张 PostgreSQL 表。

核心数据结构

createSurveyForm() 定义完整 form shape，各 section 组件直接绑定对应分段。

配置方式

API 地址和数据库连接走环境变量，模板在 .env.example。

4.1 App.vue - 表单状态与提交入口 
const form = reactive(createSurveyForm());
const isSubmitting = ref(false);

async function handleSubmit() {
  if (validationErrors.value.length > 0) {
    alert(`Please complete required fields:\n\n${validationErrors.value.join("\n")}`);
    return;
  }

  if (isSubmitting.value) return;
  isSubmitting.value = true;

  try {
    const result = await submitSurvey(form);
    alert(`Questionnaire saved successfully: ${result.survey_id}`);
  } catch (err) {
    console.error(err);
    alert(`Failed to save questionnaire: ${err.message}`);
  } finally {
    isSubmitting.value = false;
  }
}

4.5 surveyApi.js - 前端提交 API 
const API_BASE_URL = import.meta.env.VITE_API_BASE_URL || "http://localhost:3000";

export async function submitSurvey(form) {
  const response = await fetch(`${API_BASE_URL}/api/surveys`, {
    method: "POST",
    headers: {
      "Content-Type": "application/json"
    },
    body: JSON.stringify(form)
  });

  const contentType = response.headers.get("content-type") || "";
  const result = contentType.includes("application/json")
    ? await response.json()
    : { message: await response.text() };

  if (!response.ok) {
    throw new Error(result.message || "Failed to submit survey");
  }

  return result;
}

      survey_id: basic.survey_id
    });
  } catch (err) {
    await client.query("ROLLBACK");

5. 数据流说明

用户填写表单
  -> 各章节组件通过 v-model 写入 form.section_x
  -> App.vue 的 validationErrors 检查必填和范围
  -> submitSurvey(form) 发送 POST /api/surveys
  -> server.js 解构 basic / section_a / section_b ...
  -> PostgreSQL transaction: BEGIN
  -> 多张表 INSERT
  -> COMMIT 成功，或 ROLLBACK 回滚
  -> 前端弹出成功/失败提示

8. 后续可继续优化

  • 把 server.js 拆成 db.js、validators.js、surveyRepository.js。
  • 后端补更严格的字段校验，例如 Likert 只能是 1 到 5，经纬度必须在合法范围。
  • 给提交成功后的交互加确认状态，例如保存后重置表单或自动生成下一份 Survey ID。
  • 如果后续做 AI 填表，建议新增后端接口，不要在浏览器暴露 API Key。
